# P2P Planner v1.0.0-beta.2 release notes draft

- Release type: GitHub Pre-release / beta
- Tag: `v1.0.0-beta.2`
- Previous public beta: `v1.0.0-beta.1`
- Release readiness basis: `release-gates --profile full-local-release` passed on 2026-06-04 with score `89/100`, raw class `beta_candidate`, effective class `internal_candidate` because repeatability is not yet fully proven.

## Summary

`v1.0.0-beta.2` is the second beta release candidate for the web-first self-hosted P2P Planner line. It keeps the product promise focused: a usable Kanban core with auth/session, workspaces, boards, columns, cards, card details, appearance customization, activity/history surfaces, local-first runtime baseline, sync baseline and export/backup preview.

The main difference from the first beta line is release evidence maturity. The current repository can now pass a full local release-gates profile with managed PostgreSQL, managed backend/frontend runtime, repeated backend smoke, frontend build/tests, UIX mocked flow, UIX real-backend core flow, browser smoke and clean-machine sandbox.

## Included user-facing scope

- Sign up, sign in, refresh session, sign out and sign out-all.
- Workspace, board, column and card CRUD.
- Card details drawer with title, description, status, priority, dates, labels, checklists and comments.
- Drag-and-drop card movement/reorder.
- User appearance and board appearance settings.
- Board activity, card activity and workspace audit-log surfaces.
- Local-first visible state baseline for board/card runtime.
- Backend sync baseline: replica registration, push/pull cursor and tombstone-aware core events.
- Board/workspace backup export and non-destructive import preview.

## Known limitations

See `docs/product/v1-known-limitations.md`. The important beta.2 limitations are:

- release is still beta/pre-release, not stable `v1.0.0`;
- repeatability hard cap must be lifted before calling the release stable;
- P2P/relay/mobile are out of v1;
- import execution remains non-destructive preview only;
- full conflict-resolution UI and E2EE are not included;
- Linux AppImage and Windows executable assets must be smoke-tested as release artifacts before upload.
